# Discord Bot для уведомлений о событиях

Бот для Discord с автоматическими уведомлениями о событиях по расписанию.

## Возможности

- Автоматические уведомления в указанное время (МСК)
- Напоминания администратору в ЛС за 2 часа и 30 минут до события
- В точное время — ЛС всем у кого есть упомянутые роли (один раз на пользователя)
- Embed-сообщения с упоминанием ролей и счётчиком участников по ролям
- Кнопка «Присоединиться» с подсчётом участников и списком в embed
- **Автоудаление сообщения в канале через 1 час** (тестовые — через 5 минут)
- Удаления переживают рестарт (план хранится в SQLite)
- Хранение настроек, участников и расписаний удаления в SQLite
- Персистентные slash-команды и UI
- Готов к деплою на Railway

## Установка

1. Клонируйте репозиторий
2. Установите зависимости:
```bash
pip install -r requirements.txt
```

3. Создайте файл `.env` (см. `.env.example`) и добавьте токен бота:
```
DISCORD_TOKEN=your_bot_token_here
```

## Privileged Intents

Бот использует **привилегированные интенты**. Их нужно включить в Discord Developer Portal:

1. Откройте https://discord.com/developers/applications
2. Выберите своё приложение → **Bot**
3. Включите:
   - **Server Members Intent** — нужен для подсчёта участников по ролям
   - **Message Content Intent** — оставлен включённым для совместимости

## Использование

### Команда /setup

Настройте уведомления с помощью слеш-команды:

```
/setup time:15:00 channel:#уведомления event_name:capt
```

Параметры:
- **time** — время по МСК, выбирается из списка: `12:00`, `13:00` … `22:00` (с автокомплитом)
- **channel** — канал для отправки уведомления (нативный пикер канала)
- **event_name** — название события из списка (`capt`)
- **admin** *(опционально)* — пользователь-администратор для ЛС-напоминаний. Если не указан, используется тот, кто вызвал команду.

После отправки `/setup` бот покажет эфемерный **RoleSelect** — там можно выбрать **от 1 до 25 ролей** одним кликом, без копирования ID.

Чтобы расширить список времён или названий событий — отредактируйте `TIME_CHOICES` / `EVENT_NAME_CHOICES` в начале `main.py`.

### Команда /test

Отправить тестовое уведомление в текущий канал:

```
/test
```

Проверяет как будет выглядеть embed и работает ли кнопка «Присоединиться».

### Расписание

События происходят:
- Понедельник
- Среда
- Пятница
- Суббота

В указанное время по **МСК** (Europe/Moscow).

### Уведомления

1. **За 2 часа до события** — бот пишет в ЛС администратору.
2. **За 30 минут до события** — бот снова пишет в ЛС.
3. **В точное время** — бот:
   - отправляет embed в канал с упоминанием ролей и кнопкой «Присоединиться»;
   - рассылает ЛС всем у кого есть любая из упомянутых ролей (один раз на пользователя, даже если у него несколько таких ролей) + админу;
   - **через 1 час автоматически удаляет это embed-сообщение** из канала, чтобы не мусорил.

### Команда /test_dm

Принудительно отправить ЛС всем участникам с упомянутыми ролями + админу — для проверки логики ЛС без ожидания расписания.

```
/test_dm
```

### Автоудаление

- Основное сообщение события — удаляется через **1 час**.
- Сообщение от `/test` — удаляется через **5 минут**.
- Расписание удаления хранится в SQLite, поэтому переживает перезапуск бота.

⚠️ Боту нужно право **Manage Messages** в канале, чтобы удалять свои сообщения (для своих сообщений права обычно достаточно стандартного `Send Messages`, но если канал ограничен — нужен `Manage Messages`).

### Кнопка «Присоединиться»

- Клик по кнопке добавляет пользователя в список участников **на этот день**.
- Повторный клик — убирает из списка (toggle).
- Список участников и счётчик отображаются в embed-сообщении и обновляются в реальном времени.
- Каждый день список начинается заново.

## Деплой на Railway

1. Создайте новый проект на Railway.
2. Загрузите файлы проекта (или подключите Git).
3. В **Variables** добавьте `DISCORD_TOKEN` со значением токена бота.
4. Railway автоматически прочитает `Procfile` и запустит worker.

⚠️ Для сохранения базы между редеплоями примонтируйте Railway Volume к директории с `bot_config.db` (или используйте PostgreSQL).

## Структура проекта

- `main.py` — основной файл бота
- `requirements.txt` — зависимости Python
- `Procfile` — конфигурация для Railway
- `.env.example` — пример переменных окружения
- `bot_config.db` — база данных SQLite (создаётся автоматически)

## Получение ID

Включите Developer Mode в Discord: **Settings → Advanced → Developer Mode**.

- **ID роли** — правый клик на роль → Copy ID
- **ID канала** — правый клик на канал → Copy ID
- **ID пользователя** — правый клик на пользователя → Copy ID
